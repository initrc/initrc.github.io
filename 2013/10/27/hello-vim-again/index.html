Hello Vim, Again

2013.10.27

After years of use, I’m still excited to discover new things in vim.

Why Vim

Keyboard

It all starts with the keyboard, or laziness. Why stop typing, move your hand over to the mouse, move and click the mouse, and move your hand back and continue typing? Context switching is expensive. No moving over to the arrow keys? That’s even better.

To back myself a bit, I audited an User Interfact Design course for the first two weeks of a semester. No, it was not fun enough to take, but I do remember a chapter in the textbook described that everything in user interaction comes with a cost. A keystroke, a combination keystroke, a mouse move, a mouse click, double click, click with a key down, click before/after keystrokes, hands movement, etc. They all come with different costs, which can be quantified in general. Personally, I feel hand movements cost way more than keystrokes.

Yeah, I’m also a happy vimium user.

Configurable

With plugins and vimrc, vim can be personalized while still stay lightweight and fast. Although vim was introduced in 1991, plugin development is still active today.

Terminal Integration

It works in a terminal, and even better, in a terminal multiplexer like the awesome Tmux. So you can use vim remotely without losing any efficiency.

More

It’s fast.
It’s free.
It’s two characters to launch.
…

Why Learn Vim Again

I used to think that NERDTree and powerline were the only plugins I would ever need. The rest of this article proves how wrong that was.

Plugins

Plugin Management

Vundle

Plugin management was never easy in vim. I guess that’s why I kept it simple and only installed two plugins. Pathogen created a file structure convention to organize plugins, and Vundle came in later and made it really easy to install, update and remove plugins. All plugin code live in .vim/bundle, which keeps your .vim folder neat. No more plugin code in the dotfiles repo, and no more git submodules.

Code Completion

YouCompleteMe

Learn the API, learn the codebase, and never use code completion. That’s what I do. Sometimes looking for an auto-completed word takes more brain power than just typing it. Plus, when I first learned Java, code completion was super slow in eclipse. First impression matters.

But I still wanted to know if it works in vim so I tried YouCompleteMe. It’s fast and generally clean. However, it does have some obtrusive UI that I don’t like. But they’re configurable.

YouCompleteMe works well with C family languages and Python out of the box. It even jumps to definition/declaration!

It can be hard to install though. If you hit any ABRT or python error, reinstall python, than vim (either MacVim or vim with python support), and finally YouCompleteMe.

Eclim

To get Java working you need Eclim, which plays well with YouCompleteMe. It basically starts a daemon process that utilizes eclipse for code completion and error checking. Unfortunately, no jump to definition/declaration.

To use it you either need to start eclipse and open the eclim window or start the daemon with $eclipse_home/eclimd & > /dev/null 2>&1.
Then create some eclipse files in your project with :CreateProject <path> -n java. It also works with ruby.
Finally use it as omnifunc, which is picked up by YouCompleteMe.

I created this eclim-vundle repo to mirror eclim (currently 2.3.2) so that you can skip the vim part in Eclim installation and let Vundle manages it. Otherwise, be sure not to install it to ~/.vim ‘cause it will mess up your .vim folder again. Choose ~/.vim/bundle/eclim/ instead.

Tried YouCompleteMe and Eclim for a day, not my cup of tea. I do like the jump-to-definition feature though.

Fancy Status

Vim-airline

Pure vimscript. Way less code than powerline and vim-powerline. Apart from all the fancy status and integration with other plugins, a great thing about the status plugin is that you don’t have to look at it to know if you are in command mode or insert mode.

Another thing I like is if the filepath is super long, vim-airline shows the last part, which includes the filename. Powerline shows from the beginning and truncates the rest.

Grab prepatched fonts at powerline-fonts. Highly recommend DejaVu Sans Mono and Meslo.

File Navigation

NERDTree

My favorite file manager. I wish one day vim will support multiple fonts so that NERDTree can use non-mono fonts to display filenames. Huge space saving there.

Some configuration to open NERDTree when opening vim in a folder, and exits when closing the last editing window.

Ctrlp

Awesome pure vimscript plugin. Command-T is also great, but too much work to install.

I love the working path mode of ‘ra’. ‘r’ finds the nearest ancestor that contains .git/.svn/etc.

Format

Neinspace

I’m not a fan of formatters because they are scary. But sometimes I need them to clean the code with absolutely valid rules. So I created my own vim plugin to reveal and get rid of whitespace, tabs, carriage return, etc. Here’s a post introducing Neinspace.

Git

Vim-fugitive

I love scm_breeze and never ran git commands in vim, until I picked up vim-fugitive. :Gblame is pretty useful.

Run Code

Runner

I created Runner so that I don’t have to jump to another tmux pane to run some simple code, although it’s only 2.5 keys away to switch (Ctrl-a, j), and 2 keys to run the code (up and enter, if ran before). Well, you have to move your hands to the right though, and it’ll be worse if the command to run the code changes.

Runner compiles/interprets single files. The supported languages include Java, C, C++, Python, and Ruby. One key to run them all! Perfect for code challenges.

Search

Ag

Super fast! Just type :Ag followed by the keyword.

Syntax

Syntastic

It works for almost all languages, but you need to install checkers yourself. I mainly use it for Python.

Mapping

The goal is obvious: whatever makes you fast. Esc, F key, even \ are too far away.

End

Configure your vim well and configure it early. But don’t be obsessed with it. Let it be your tool and happy coding.
